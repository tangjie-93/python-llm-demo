# 06. 术语表

## Agent

能基于目标、上下文和工具执行多步任务的 AI 系统。Agent 不等于聊天机器人，它通常能调用工具、管理状态，并根据观察结果继续决策。

## Agent Loop

模型反复执行“思考、选择工具、执行工具、观察结果、继续决策”的循环，直到任务完成、失败或等待人工确认。

## Tool Calling

模型根据工具 schema 生成工具名称和结构化参数，程序负责校验参数并执行真实工具。

## Structured Output

要求模型按 JSON Schema 或固定结构输出，便于前后端、工作流和评测系统稳定解析与校验。

## RAG

Retrieval-Augmented Generation，检索增强生成。先检索外部知识，再把证据注入上下文生成回答。

## Embedding

把文本转换为向量，用于语义搜索、聚类、去重和相似度计算。

## Vector Database

用于存储向量并执行相似度搜索的数据库，例如 pgvector、Qdrant、Milvus、Weaviate。

## BM25

传统关键词检索算法，适合精确术语、编号、错误码、API 名称等查询。

## Hybrid Search

向量检索和关键词检索结合，提高召回质量。

## Rerank

对初步检索结果重新排序，提升最终上下文相关性。

## Context Window

模型一次请求可以看到的上下文长度，包括用户输入、系统指令、历史消息、工具结果和 RAG 内容。

## Context Engineering

围绕上下文选择、压缩、排序、隔离和注入进行系统设计，目标是让模型看到足够、可信、相关的信息。

## Hallucination

模型生成没有依据或错误的内容。

## Human-in-the-loop

在 Agent 执行关键步骤时引入人工确认、修改、审批或接管。

## Checkpoint

长任务执行中的状态保存点，用于恢复、中断、重试和审计。

## Tracing

记录模型输入输出、工具调用、检索结果、耗时、成本和错误的完整链路。

## Eval

用数据集和指标评估 Agent、RAG、Prompt 或工具调用质量。可以评估答案、检索、工具调用、执行轨迹和安全性。

## Golden Dataset

人工标注或人工确认过的标准样例集，用于回归测试和质量对比。

## Prompt Injection

用户或外部内容试图覆盖系统规则、诱导泄露信息或执行危险操作。

## MCP

Model Context Protocol，用统一协议把工具、资源和提示暴露给模型应用。

## Guardrail

防止输入、输出、工具调用和权限边界越界的规则、校验和控制机制。

## Sandbox

隔离执行环境，用于限制代码、文件、网络和系统资源访问。
