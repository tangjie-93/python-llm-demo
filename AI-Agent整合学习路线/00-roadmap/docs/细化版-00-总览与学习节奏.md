# 00. 总览与学习节奏

## 转型目标

AI Agent 开发的核心是让大模型在受控环境里完成任务。它通常包含：

- 模型：理解用户意图、生成计划、选择工具、总结结果
- 工具：API、数据库、文件系统、搜索、浏览器、业务系统
- 状态：当前任务进度、历史上下文、用户确认、失败重试
- 权限：谁能调用什么工具，什么操作必须审批
- 评测：如何证明 Agent 变好了，而不是凭感觉
- 交互：如何让用户理解 Agent 正在做什么，并能随时接管

你作为 Vue 前端工程师，需要重点建立“AI 应用工程能力”，而不是优先学习深度学习理论。

## 每天 2 小时学习节奏

### 30 分钟：读官方文档

优先读官方文档，避免被二手文章误导。重点不是背 API，而是理解概念边界：

- 这个能力解决什么问题
- 它不适合什么场景
- 它的失败模式是什么
- 如何观测和测试

### 60 分钟：写代码

每个知识点都应该落到代码里。建议每周至少完成一个小功能：

- 一个工具函数
- 一个 Agent Loop
- 一个 RAG 检索策略
- 一个 Vue 交互组件
- 一个 eval case

### 20 分钟：记录失败案例

Agent 开发的成长很大一部分来自失败案例积累：

- 模型选错工具
- 参数生成错误
- 检索结果无关
- 工具返回太长
- 用户意图误解
- 权限边界不清
- 流式 UI 状态错乱

### 10 分钟：整理笔记

建议固定记录：

- 今天学了什么
- 哪个概念还不清楚
- 代码跑通了什么
- 失败原因是什么
- 明天要验证什么

## 判断自己是否掌握的标准

不要用“看懂文档”作为掌握标准。更可靠的标准是：

- 能手写一个最小实现
- 能解释这个能力的失败模式
- 能给它加测试或 eval
- 能把它接到一个 Vue 页面
- 能说清楚生产环境的权限和成本问题
