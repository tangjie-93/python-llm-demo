# 01. 24 周阶段学习计划

## 第 1-4 周：Python 后端基础

| 周 | 主题 | 核心知识点 | 产出 |
| --- | --- | --- | --- |
| 1 | Python 语法速成 | 类型、函数、类、模块、装饰器、CLI、Pythonic 写法 | Markdown 字数统计 CLI |
| 2 | Python 工程化 | uv / pip、虚拟环境、包结构、pytest、asyncio、httpx | 异步 API 调用工具 |
| 3 | FastAPI 框架 | 路由、Pydantic、依赖注入、中间件、异常处理、JWT | 笔记 API 骨架 |
| 4 | 数据库与项目交付 | SQLAlchemy、Alembic、PostgreSQL、CRUD、认证、测试 | `dev-note-api` |

验收：能独立写一个带认证、数据库、测试、文档的 FastAPI 后端。

## 第 5-7 周：LLM 基础与 API 调用

| 周 | 主题 | 核心知识点 | 产出 |
| --- | --- | --- | --- |
| 5 | LLM 核心概念 | Token、Tokenizer、Embedding、Transformer、模型选型、上下文窗口 | Token / Embedding 实验 |
| 6 | LLM API 实战 | OpenAI SDK、temperature、top_p、streaming、多轮对话、错误处理 | 终端 ChatGPT |
| 7 | 智能代码审查 CLI | System Prompt、文件分析、Git Diff、结构化输出、报告格式化 | `ai-code-review` |

验收：能解释模型参数影响，能处理流式输出、错误重试、多模型配置和结构化结果。

## 第 8-11 周：Prompt 工程与 RAG

| 周 | 主题 | 核心知识点 | 产出 |
| --- | --- | --- | --- |
| 8 | Prompt 工程 | 角色设定、少样本、约束、输出格式、Prompt 版本管理 | Prompt 模板库 |
| 9 | RAG 核心 | 文档解析、chunking、embedding、向量库、检索、citation | 最小 RAG 问答 |
| 10 | LangChain / LlamaIndex | Loader、Splitter、Retriever、Chain、Index、Query Engine | 框架版 RAG |
| 11 | 文档问答系统 | 上传、索引、检索、回答、引用、拒答、追问 | `doc-qa` / Vue 知识库 Agent |

验收：答案必须有证据，资料缺失时能拒答，能比较 chunk、hybrid search、rerank 对效果的影响。

## 第 12-16 周：Agent 框架与多步骤任务

| 周 | 主题 | 核心知识点 | 产出 |
| --- | --- | --- | --- |
| 12 | Agent 核心 | Agent Loop、ReAct、Tool Calling、工具参数 schema、多工具协同 | 天气 / 计算 / 搜索 Agent |
| 13 | LangGraph 工作流 | StateGraph、节点、边、路由、checkpoint、interrupt / resume | 可恢复对话系统 |
| 14 | Tool 与 Memory | 工具设计、工具注册、短期记忆、长期记忆、摘要记忆 | 带 Memory 的 Agent |
| 15 | Multi-Agent | Supervisor、Planner / Executor / Reviewer、CrewAI、AutoGen 边界 | 多 Agent 协作 Demo |
| 16 | 智能客服系统 | 意图识别、知识库、工单、人工转接、前后端整合 | `agent-support` |

验收：能设计状态机、恢复中断任务、解释什么时候该用 workflow，什么时候该用 agent。

## 第 17-19 周：MCP 协议与工具生态

| 周 | 主题 | 核心知识点 | 产出 |
| --- | --- | --- | --- |
| 17 | MCP 概念 | Host、Client、Server、Tools、Resources、Prompts、stdio / HTTP 传输 | 使用现有 MCP Server |
| 18 | 自定义 MCP Server | 工具定义、参数校验、资源暴露、错误返回、Inspector 调试 | GitHub / 文件系统 MCP |
| 19 | Agent + MCP 集成 | 工具权限、tool choice、业务 API 封装、审计日志 | 企业工具 Agent |

验收：能把业务系统包装成 MCP Server，并在 Agent 中受控调用。

## 第 20-22 周：工程化、部署与质量

| 周 | 主题 | 核心知识点 | 产出 |
| --- | --- | --- | --- |
| 20 | Observability 与 Eval | tracing、日志、golden dataset、retrieval eval、answer eval、trajectory eval | Eval Dashboard |
| 21 | Docker 与 CI/CD | Dockerfile、Compose、环境变量、迁移、GitHub Actions | 可部署服务 |
| 22 | 生产化 | RBAC、sandbox、PII 脱敏、rate limit、成本控制、监控告警 | 上线 Checklist |

验收：能回答“Agent 有没有变好”“哪里失败了”“一次请求花多少钱”“谁调用了什么工具”。

## 第 23-24 周：作品集整合

把前面能力整合为一个可展示项目，优先选择：

- AI Vue 前端工程 Agent
- 企业知识 + 操作 Agent
- AI 数据分析 Agent

最终交付应包含：在线演示、README、架构图、核心代码、eval 报告、tracing 截图、安全说明、部署文档。
