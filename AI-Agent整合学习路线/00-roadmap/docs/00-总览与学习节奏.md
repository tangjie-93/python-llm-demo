# 00. 总览与学习节奏

## 转型目标

AI Agent 开发不是简单调用大模型 API，而是让模型在受控环境中可靠完成任务。核心系统包括：

- 模型：理解意图、生成计划、选择工具、总结结果。
- 工具：API、数据库、文件系统、搜索、浏览器、CRM / OA 等业务系统。
- 状态：任务进度、上下文、记忆、用户确认、失败重试、中断恢复。
- 权限：工具访问控制、敏感操作审批、沙箱、审计日志。
- 评测：用 eval、tracing、数据集和线上指标证明 Agent 质量。
- 交互：让用户看懂 Agent 正在做什么，并能确认、撤销、接管。

对前端 / Vue 工程师来说，重点是建立 AI 应用工程能力：把 Agent 做成真实产品，而不是只学习深度学习理论。

## 每天 2 小时学习节奏

如果每天只能投入 2 小时，可以按下面节奏拆分每周任务。

### 30 分钟：读官方文档

优先读官方文档，避免被二手文章误导。重点不是背 API，而是理解概念边界：

- 这个能力解决什么问题。
- 它不适合什么场景。
- 它的失败模式是什么。
- 如何观测和测试。

### 60 分钟：写代码

每个知识点都应该落到代码里。建议每周至少完成一个小功能：

- 一个工具函数。
- 一个 Agent Loop。
- 一个 RAG 检索策略。
- 一个 Vue 交互组件。
- 一个 eval case。

### 20 分钟：记录失败案例

Agent 开发的成长很大一部分来自失败案例积累：

- 模型选错工具。
- 参数生成错误。
- 检索结果无关。
- 工具返回太长。
- 用户意图误解。
- 权限边界不清。
- 流式 UI 状态错乱。

### 10 分钟：整理笔记

建议固定记录：

- 今天学了什么。
- 哪个概念还不清楚。
- 代码跑通了什么。
- 失败原因是什么。
- 明天要验证什么。

## 技术栈建议

### 前端

- Vue 3、TypeScript、Vite、Nuxt 3
- Pinia、Vue Router
- Element Plus / Naive UI / Arco Design Vue
- ECharts、Vue Flow
- markdown-it / md-editor-v3、shiki
- SSE / WebSocket 流式交互

### 后端

- Python、FastAPI、Pydantic、asyncio
- SQLAlchemy、PostgreSQL、pgvector
- Redis、Celery / RQ
- Docker、CI/CD、对象存储、任务调度

### Agent

- OpenAI API / Responses API / Agents SDK
- LangChain、LangGraph、LlamaIndex
- MCP
- LangSmith / promptfoo / 自建 eval dataset

## 掌握标准

不要用“看懂文档”作为掌握标准。更可靠的标准是：

- 能手写最小实现。
- 能解释失败模式。
- 能写测试或 eval。
- 能接到 Vue 页面。
- 能说清楚权限、成本、观测、部署方案。
