# 09. 术语表

## Agent

能根据目标、上下文和工具集合，自主选择下一步动作的软件系统。Agent 不等于聊天机器人，它通常能调用工具、管理状态并完成多步骤任务。

## Agent Loop

Agent 的循环执行过程：模型思考、选择工具、执行工具、观察结果、继续下一步，直到完成任务或停止。

## Tool Calling

模型根据工具 schema 生成调用意图和参数，程序负责真正执行工具。

## Structured Output

让模型按照固定结构输出，例如 JSON。用于保证前后端和评测系统可以稳定解析。

## RAG

Retrieval-Augmented Generation，检索增强生成。先从外部资料中检索相关内容，再让模型基于这些内容回答。

## Embedding

把文本转换成向量，用于语义搜索、聚类和相似度计算。

## Vector Database

支持向量存储和相似度搜索的数据库，例如 pgvector、Qdrant、Milvus。

## BM25

传统关键词检索算法，适合精确术语、编号、错误码等查询。

## Hybrid Search

结合向量检索和关键词检索的方法，通常比单一检索方式更稳。

## Rerank

对初步召回的搜索结果重新排序，提高最终放入上下文的内容质量。

## Context Window

模型一次请求可以看到的上下文长度，包括用户输入、系统指令、历史消息、工具结果和 RAG 内容。

## Context Engineering

设计模型输入上下文的工程方法，包括选择哪些信息、如何排序、如何压缩、如何隔离不可信内容。

## Hallucination

模型生成看似合理但没有依据或错误的内容。

## Human-in-the-loop

在 Agent 执行关键步骤时引入人工确认、修改或审批。

## Checkpoint

长任务执行中的保存点，用于恢复、中断、重试和审计。

## Tracing

记录一次 Agent 运行中的模型调用、工具调用、参数、结果、耗时、token 和错误。

## Eval

评测 Agent 质量的方法。可以评估答案、检索、工具调用、执行轨迹和安全性。

## Golden Dataset

人工标注的标准测试集，用于回归测试和质量对比。

## Prompt Injection

用户或外部内容试图覆盖系统规则或诱导模型执行危险操作。

## MCP

Model Context Protocol，用于标准化 AI 应用连接工具、资源和提示模板的协议。

## Guardrail

保护机制，用于限制输入、输出、工具调用和安全边界。

## Sandbox

隔离执行环境，用于限制代码、文件、网络和系统权限。
