# Level 3-4 | 第 11 周：项目实战 — 技术文档 RAG 问答系统

> 🏗️ **关卡名**：文档智者 · RAG 系统交付
> 📅 **时间**：第 11 周 | ⏱️ **学时**：~20h

## 本周学习目标

- [ ] 完成阶段三实战项目 `doc-qa`
- [ ] 能处理「文档中不包含此信息」的边界情况
- [ ] 能实现流式返回和来源追溯

## 每日学习安排

### 周一（4h）· 项目架构设计

- [ ] 设计：`doc-qa` 系统架构（API 层 → 检索层 → LLM 层）
- [ ] 设计：文档上传 API（`POST /documents`）
- [ ] 设计：问答 API（`POST /qa`，支持流式）

### 周二（4h）· 文档处理管道

- [ ] 实现：文档上传 → 存储 → 切分 → Embedding → 向量库
- [ ] 实现：多文档管理（文档列表、删除、状态）
- [ ] 实现：异步处理（上传后后台处理）

### 周三（4h）· 问答引擎

- [ ] 实现：用户提问 → 检索相关片段 → 注入 Prompt → LLM 生成
- [ ] 实现：来源引用返回（文档片段 + 位置信息）
- [ ] 实现：流式回答（WebSocket 或 SSE）
- [ ] 实现：「不知道」处理（检索结果相似度过低时的回复策略）

### 周四（4h）· 前端页面（可选）

- [ ] 实现：简单的问答 Web 界面（React/Vue 单页）
- [ ] 实现：流式输出渲染（打字机效果）
- [ ] 实现：来源引用高亮显示

### 周五（4h）· 测试 + 文档

- [ ] 测试：多文档问答准确性
- [ ] 测试：边界情况（空文档、超大文档、无结果）
- [ ] 编写 README
- [ ] 对照验收标准自检

## 本周产出

- ✅ **`doc-qa`** 完整的 RAG 问答系统
- ✅ API 端文档上传和问答接口
- ✅ 流式返回 + 来源引用
- ✅ （可选）前端问答界面

## 通关标志

- [ ] 可通过 API 上传文档并触发向量化流程
- [ ] 问答返回的结果包含文档来源引用
- [ ] 回答质量：上下文中有的信息准确提取，没有的明确表示不知道
- [ ] 支持流式返回回答
- [ ] API 响应时间 < 5 秒（不含模型推理时间）
- [ ] 处理「文档中不包含此信息」的边界情况

## 前端技能衔接提示

- SSE/WebSocket 流式渲染 = 前端聊天应用（ChatGPT-like UI）
- 来源引用展示 = 搜索结果中的「相关片段」高亮
- 文档上传交互 = 文件上传组件 + 进度条（你写过无数遍）
- 用户体验设计 = 这是你作为前端的最大优势
