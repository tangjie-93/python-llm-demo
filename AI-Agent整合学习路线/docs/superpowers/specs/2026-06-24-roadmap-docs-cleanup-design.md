# Roadmap 文档整理设计

## 目标

让 `00-roadmap/docs` 只呈现一条按编号阅读的学习主线，消除重复入口、重复内容与不一致的命名。

## 最终结构

`00-roadmap/docs/` 保留以下文件：

- `README.md`：唯一入口，说明阅读顺序与各文档职责。
- `00-总览与学习节奏.md` 至 `06-术语表.md`：唯一的编号主线。
- `archive/`：不参与主线阅读的历史文档。

## 迁移规则

保留现有 `00`–`06` 内容作为权威版本。将以下历史或辅助文件移入 `archive/`：

- `细化版README.md`
- `细化版-00-总览与学习节奏.md`
- `细化版-08-学习资源索引.md`
- `细化版-09-术语表.md`
- `AI-Agent原README.md`
- `阶段学习包说明.md`
- `99-来源映射.md`

新建的 `README.md` 提供明确的 `00 → 06` 链接，并说明历史文件仅供追溯。原文件内容不修改，移动后保持可访问。

## 校验

完成后验证：根目录只含 `README.md`、`00`–`06` 和 `archive/`；README 中所有主线链接存在；归档文件均已迁移且无遗漏。
